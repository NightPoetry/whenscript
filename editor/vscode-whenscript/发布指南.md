# 发布指南:把 WhenScript 扩展传上 VS Code Marketplace

这份文档只讲"怎么把 `.vsix` 传上 Marketplace 让所有人 `ext install` 装到",不讲怎么开发扩展(那部分看 `README.md`)。跟着走一遍,从零账号到线上可见大概 15-20 分钟(不算账号审核等待)。

## 前置检查

- `package.json` 里 `"publisher": "whenscript-lang"` —— 这个名字必须跟你在 Marketplace 上创建的 publisher **一模一样**(大小写、连字符都要对上),否则发布会直接报错。如果你要用别的 publisher 名字,先改这里的值,再重新打包(见步骤 ⑤)。
- `"version"` 已经是 `0.2.0`,比之前装过的 `0.1.0` 新——Marketplace 不允许发布一个已存在的版本号,每次发布前都要把这个数字往上调。
- `icon`、`license`、`categories` 等元数据已经补齐(本次改动的内容),不用再管。

## ① 注册 Azure DevOps 账号 + 创建 Personal Access Token(PAT)

VS Code Marketplace 的发布系统架在 Azure DevOps 上,所以第一步不是去 marketplace.visualstudio.com 注册,是先有一个 Azure DevOps 账号。

1. 打开 https://dev.azure.com ,用你的 Microsoft 账号(没有就顺手注册一个,免费)登录,走一遍首次进入的组织创建向导(随便起个组织名,不影响发布,发布是挂在 publisher 上不是挂在 Azure 组织上)。
2. 登录后点右上角头像旁边的 **User settings**(小齿轮图标)→ **Personal access tokens**。
3. **New Token**:
   - **Name**:随便起,例如 `whenscript-marketplace-publish`。
   - **Organization**:选 **All accessible organizations**(重要,选错某个具体组织可能导致发布时鉴权失败)。
   - **Scopes**:点 **Show all scopes**,找到 **Marketplace**,勾 **Manage**(注意不是 Publish/只读那档,要 Manage 才能创建/管理 publisher 和发布扩展)。
   - Expiration 按需选(最长通常 1 年),到期前记得重新生成。
4. 创建后**立刻复制 token 存好**(只显示这一次,关掉页面就再也看不到明文了,只能重新生成)。这个 token 后面步骤 ③④ 都要用。

## ② 创建 publisher

`npx vsce create-publisher` 这条老命令已经废弃(vsce 新版本不再支持命令行创建 publisher),现在只能走网页:

1. 打开 https://marketplace.visualstudio.com/manage ,用同一个 Microsoft 账号登录。
2. **Create publisher**。
3. **Publisher ID** 必须填 **`whenscript-lang`**(跟 `package.json` 里的 `publisher` 字段完全一致,这是扩展 ID 的一部分,发布后不可改名,只能新建一个再迁移)。
4. **Publisher display name** 随意,例如"WhenScript"。
5. 其它资料(网站、邮箱等)按需填,不影响能否发布。

如果 `whenscript-lang` 这个 ID 已经被别人占用(概率很低但可能),那就只能改名:同时改 `package.json` 的 `publisher` 字段成新名字,然后重新走本文档步骤 ⑤ 重新打包。

## ③ 本地登录 publisher

```bash
cd editor/vscode-whenscript
npx @vscode/vsce login whenscript-lang
```

会提示你粘贴 PAT(就是步骤 ① 复制的那串),粘贴回车即可。登录状态存在本地(`~/.vsce` 之类),同一台机器以后不用重复登录,除非 token 过期或换机器。

## ④ 发布

已经打好的包在本目录:`whenscript-0.2.0.vsix`。两种发布方式:

**方式一,直接发布已打好的 vsix(推荐,发布的就是你验证过的那个包)：**

```bash
npx @vscode/vsce publish --packagePath whenscript-0.2.0.vsix
```

**方式二,让 vsce 重新打包并直接发布（跳过手动 package 那一步）：**

```bash
npx @vscode/vsce publish
```

**如果没有提前 `login`(步骤 ③),也可以带着 token 一次性发布：**

```bash
npx @vscode/vsce publish -p <你的PAT>
```

发布成功后 Marketplace 页面地址是:

```
https://marketplace.visualstudio.com/items?itemName=whenscript-lang.whenscript
```

首次发布到能在 VS Code 内搜到,通常几分钟到十几分钟内生效(有索引延迟,不是发布命令返回就立刻全球可搜)。

## ⑤ 后续升级发布

每次改完代码要发新版本:

1. 改 `package.json` 的 `"version"`(遵循语义化版本,小改 `0.2.0`→`0.2.1`,加功能 `0.2.0`→`0.3.0`)。
2. 在 `CHANGELOG.md` 顶部加一条新版本记录。
3. 重新打包验证:
   ```bash
   cd editor/vscode-whenscript
   npx @vscode/vsce package --allow-missing-repository
   ```
4. 确认没问题后发布:
   ```bash
   npx @vscode/vsce publish --packagePath whenscript-<新版本号>.vsix
   ```

也可以用 `npx @vscode/vsce publish patch`(或 `minor`/`major`)让 vsce 自动帮你把版本号 `+1` 并直接发布,省得手动改 `package.json`——但这样版本号变化不会自动写进 `CHANGELOG.md`,记得手动补一条。

## 本地安装验证(发布前 / 发布后都能用)

不经过 Marketplace,直接从本地 vsix 装到自己的 VS Code 里试:

```bash
code --install-extension editor/vscode-whenscript/whenscript-0.2.0.vsix
```

装完重启 VS Code(或命令面板 "Developer: Reload Window"),开一个 `.when` 文件确认语法高亮 / 代码片段生效。卸载:

```bash
code --uninstall-extension whenscript-lang.whenscript
```

发布到 Marketplace 之后,也可以用最终用户的路径验证——VS Code 内 `Ctrl/Cmd+Shift+X` 搜 "WhenScript" 装,或命令行:

```bash
code --install-extension whenscript-lang.whenscript
```

## 常见报错

- **`Error: Personal Access Token verification failed` / 401**:PAT 权限不够(没勾 Marketplace → Manage)、PAT 已过期,或者创建 PAT 时 Organization 没选 "All accessible organizations"。回步骤 ① 重新生成一个。
- **`Error: Publisher 'xxx' not found` 或发布时提示 publisher 不匹配**:`package.json` 里的 `publisher` 字段跟你在 marketplace.visualstudio.com/manage 创建的 Publisher ID 对不上(哪怕只差大小写)。改一个跟另一个对齐,改完 `package.json` 要重新打包。
- **`Error: You need to provide a repository URL in package.json` 之类的仓库检查报错**:本项目没有对外公开仓库,`package.json` 里故意不写 `repository` 字段,打包/发布命令都要带上 `--allow-missing-repository`,忘记带这个参数就会报这个错。
- **`Error: Version number already exists`**:这个版本号已经发布过了,改大 `version` 再重新打包发布(见步骤 ⑤)。
- **图标不显示 / 显示裂图**:确认 `package.json` 的 `"icon"` 字段指向的相对路径文件确实存在且不在 `.vscodeignore` 排除列表里(用 `npx @vscode/vsce ls` 检查打包清单里有没有 `icon.png`)。
