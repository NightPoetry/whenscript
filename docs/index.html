philosophy · 三条信号不是框架给你的反应式,是语言本身的语义。
CH1 · 事件一切皆事件赋值是事件,点击是事件,定时器到点是事件。event<T> 是一等值:可传递、可跨模块共享委托。引擎里没有一行轮询。
CH2 · 电路组合逻辑电路模型派生值用 := 持续绑定,像门电路接线。每次写入先让所有派生 settle 到不动点,再触发 when —— 读到的永远是一致的世界(glitch-free)。
CH3 · 并发透明并发写代码像单线程,引擎自动并行执行读写集不冲突的反应,结果与串行完全一致 —— 多核加速是实测过的,不是白皮书里的。
:= · 持续绑定数学学生等了很久的那个操作符= 是「此刻拷贝一个值」;:= 是「永远等于这个表达式」。像在草稿纸上写公式那样声明变量之间的关系,然后让值自己流动 —— 你只写关系,引擎负责让它一直成立。

WHENSCRIPT · := 写入即触发
number r = 5;
number area := 3.14159 * r * r;
number perim := 2 * 3.14159 * r;
// r = 10 → area、perim 自动重算,
// 你不用通知任何人。
强大,但有边界 —— 它是电路,不是魔法每次写入,引擎先让所有 := 派生值收敛到不动点(glitch-free),再往下走 —— 你读到的永远是一个前后一致的世界,不会出现「半更新」的中间态。万一你写出循环反馈 a := b + 1; b := a + 1,引擎不会无限转圈:它当场报一个 Reactive cycle,指出哪里短路了。强大是收敛的,不是失控的 —— 这正是把电路模型搬进语言的意义。
before / after同一个需求,两种人生「count 变了,把它的双倍显示出来」—— 在事件回调的世界里,你负责记得通知所有人;在 WhenScript 里,依赖关系就是语言结构。
JAVASCRIPT · 手动流水线
let count = 0, double = 0;
const listeners = [];
function setCount(v) {
count = v;
double = count * 2; // 别忘了同步派生值
for (const fn of listeners) fn(); // 别忘了通知所有人
}
WHENSCRIPT · 写入即触发
number count = 0;
number double := count * 2; // 派生:自动跟着变
when(add) { count = count + 1; }
changed(count) {
setText(label, "${count} / ${double}");
}
没有 setState、没有订阅列表、没有「忘了通知」这种 bug 类别 —— 写入即触发是语义,不是纪律。
capabilities一门完整的语言,不是一个 DSL
强类型 + 类型转换格int < float < 高精度的精度阶。计算「升-算-降」不打扰你,赋值是唯一强制点;还有 number<string>:2 定点小数这样的冒号配置位。
反应式全家族when 边沿触发、changed 每变必报、once 一次性、until 声明式生命周期、:= 持续绑定 —— 按需选择,语义各就各位。
实体与确定性回收group 圈定实体作用域,离开即回收,watcher 随作用域确定性销毁。没有 GC 暂停,也没有内存泄漏式订阅。
前端容器 + SSR 水合一切皆容器,布局就是 := 电路,没有虚拟 DOM。服务端渲染 + 客户端水合闭环,同一份 .when 前后端同构。
静默失效零容忍语法错在执行前响亮报错,语义错带行列号,非法运算给出指名道姓的警告。「改了数据界面没动」这类 bug 在引擎层面被当作最高级别的罪。
工具链全家桶check 静态诊断、fmt 保真格式化、LSP(诊断/悬停/补全)、REPL、pkg 去中心化包管理、graph 运行时依赖图,外加 VS Code 扩展。
